## 简介

本工具是一款基于 **GB/T 2828.1-2012**（等同采用 ISO 2859-1:1999）标准的**计数抽样检验方案快速查询工具**，面向质量工程师、生产管理人员及供应商审核人员，用于在来料检验、过程检验、成品检验等场景中快速确定一次抽样方案。

所有标准数据表（样本量字码表、主抽样表）均**完整内置于页面代码**中，无需联网，无需安装任何软件，打开浏览器即可使用。

---

## 功能特性

### 🔍 核心查询能力
- 根据**批量 N**、**检验水平**、**接收质量限（AQL）**、**检验严格度**四个参数，一键计算一次抽样方案
- 自动实现标准规定的**箭头跳转规则**：当字码行与 AQL 列交叉格为箭头（↑↓）时，自动向上/向下遍历至最近有效方案，并给出说明提示

### 📋 支持的参数范围

| 参数 | 可选项 |
|------|--------|
| 批量 N | 2 ~ 500000+ |
| 检验水平 | S-1、S-2、S-3、S-4、Ⅰ、Ⅱ（默认）、Ⅲ |
| 接收质量限 AQL | 0.010 ~ 1000（26 档标准序列） |
| 检验严格度 | 正常检验（默认）、加严检验、放宽检验 |

### 📊 输出结果
- **样本量字码**（A～R）
- **样本量 n**（最终检验数量）
- **接收数 Ac**（不合格品允许上限）
- **拒收数 Re**（触发拒批的下限）
- **判定准则**：用一句话清晰说明抽检决策逻辑
- **箭头修正说明**：当发生字码跳转时，标注原始字码与修正字码及原因

## 使用说明

### 操作步骤

```
1. 输入批量 N        → 填写本次送检批次的总数量
2. 选择检验水平      → 通常选「Ⅱ」（一般检验水平，最常用）
3. 选择 AQL 值       → 由质量合同或工艺文件规定
4. 选择检验严格度    → 初始为「正常检验」；连续不合格后转「加严」；持续合格后可转「放宽」
5. 点击「计算方案」  → 立即显示完整抽样方案
```

### 示例

| 输入参数 | 值 |
|----------|----|
| 批量 N | 1200 |
| 检验水平 | Ⅱ |
| AQL | 0.65 |
| 严格度 | 正常检验 |

**输出结果：**

| 字码 | 样本量 n | 接收数 Ac | 拒收数 Re |
|------|----------|-----------|-----------|
| J（→ L） | **200** | **1** | **2** |

> 字码 J 行 AQL=0.65 列为 ↓ 箭头，依标准自动跳转至字码 L（n=200），最终方案 Ac=1，Re=2。

**判定准则：** 从批中随机抽取 200 个样品，若发现的不合格品数 ≤ 1 个，则**接收**该批；若 ≥ 2 个，则**拒收**该批。

## 技术说明

### 内置数据表

| 数据表 | 内容 |
|--------|------|
| **Table 1** 样本量字码表 | 15 个批量区间 × 7 个检验水平，共 105 个字码 |
| **Table 2-A** 正常检验主表 | 16 字码行 × 26 AQL 列，含箭头标记 |
| **Table 2-B** 加严检验主表 | 16 字码行 × 26 AQL 列，含箭头标记 |
| **Table 2-C** 放宽检验主表 | 16 字码行 × 26 AQL 列，含箭头标记 |

### 技术栈

- **纯 HTML5 + CSS3 + 原生 JavaScript**，零外部依赖
- 无 jQuery、无 Vue/React、无 CDN 引用
- 单文件部署，离线可用
- 响应式布局，支持 PC 及移动端浏览器

### 部署方式

托管于 **GitHub Pages**，代码开源透明，自动 CI/CD，推送即发布。
